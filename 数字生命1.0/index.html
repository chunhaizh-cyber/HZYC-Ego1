核心概念

数字生命最先做的只有一件事：不停地"看"，把每一帧深度数据变成自己的"场景世界树"， 然后用这些场景去更新"安全感"和"好奇心"这两个最原始的需求。

深度感知

通过深度相机获取3D点云数据，将原始深度信息转换为统一坐标系下的空间点集。

世界树构建

将点云分割聚类，生成存在节点，构建层次化的场景世界树认知框架。

情感驱动

基于环境变化动态更新安全感和好奇心，驱动认知行为的自适应调整。

技术架构

核心类结构

ExistenceNode (存在节点)

表示场景中的物体，包含位置、尺寸、形状等几何特征

SelfNode (自我节点)

表示数字生命自身，维护安全度、好奇度等状态

StateNode (状态节点)

场景快照，包含某一时刻的所有存在和自我状态

SceneNode (场景节点)

场景容器，管理多个状态节点的时间序列

核心算法详解

点云分割算法

// 基于距离的连通性聚类
for each unprocessed point:
  create new cluster
  find all connected points (distance < threshold)
  add to cluster if size > min_size
return clusters

使用距离阈值1.0进行连通性分析，最小聚类大小为5个点， 有效分离场景中的不同物体。

安全度评估

// 多因素安全评估
min_distance = find_nearest_object()
if min_distance < 2.0: safety -= 0.2
if approaching_objects > 0: safety -= 0.1 * count
if stable_environment: safety += 0.05

综合考虑最近物体距离、靠近物体数量和场景稳定性， 实现本能的安全感评估机制。

跨帧匹配

// 基于特征相似度匹配
for each current_existence:
  best_score = calculate_similarity(position, size)
  if best_score > threshold:
    update_trajectory()
    link_existences()

通过位置和尺寸特征计算相似度，阈值0.7， 实现连续时间序列中同一物体的轨迹跟踪。

好奇度评估

// 探索驱动评估
unknown_count = count_distant_objects()
high_change = count_fast_moving_objects()
curiosity += 0.05 * unknown_count
curiosity += 0.03 * high_change
if stable: curiosity -= 0.02

基于未知区域和变化频率驱动好奇心， 鼓励系统探索新环境和适应变化。
